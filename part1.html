۱ خروجی قطعه کد رو به رو کدام است؟ 

x = 5
y = 5
z = x
print(x is y)
print(x is z)
True
True
True
False
False
True
False
False
توضیح: در پایتون، برای بهینه‌سازی، اعداد صحیح کوچک (معمولا از -5 تا 256) یک بار در حافظه ساخته می‌شوند. بنابراین `x` و `y` به یک شیء یکسان در حافظه اشاره می‌کنند و `x is y` برابر `True` است. دستور `z = x` نیز باعث می‌شود `z` دقیقا به همان شیء `x` اشاره کند، پس `x is z` هم `True` است.

۲ خروجی قطعه کد رو به رو کدام است؟ 

text = 'hello world'
new_text = text.replace('l', 'p')
print(text)
heppo worpd
hello world
hepPo worpd
error
توضیح: رشته‌ها (strings) در پایتون تغییرناپذیر (immutable) هستند. متد `.replace()` رشته اصلی را تغییر نمی‌دهد، بلکه یک رشته جدید تولید می‌کند. از آنجایی که ما در نهایت متغیر اصلی یعنی `text` را چاپ می‌کنیم، خروجی بدون تغییر خواهد بود.

۳ خروجی کد زیر چیست؟ 

set1 = {'apple', 'banana', 'cherry'}
set2 = {'google', 'microsoft', 'apple'}
set3 = set1.intersection(set2)
print(set3)
{'apple', 'banana', 'cherry', 'google', 'microsoft'}
{'banana', 'cherry', 'google', 'microsoft'}
{'apple'}
Error: Sets cannot be intersected
توضیح: متد `.intersection()` اشتراک دو مجموعه (set) را برمی‌گرداند. تنها عضو مشترک بین `set1` و `set2` رشته `'apple'` است.

۴ خروجی کد زیر چیست؟ 

thisset = {'apple', 'banana', 'cherry', 1, 0, True, False}
print(thisset)
{'apple', 'banana', 'cherry', 1, 0, True, False}
{'apple', 'banana', 'cherry', 1, 0}
error
{False, True, 'apple', 'banana', 'cherry', 0, 1}
توضیح: مجموعه‌ها (sets) اعضای تکراری را ذخیره نمی‌کنند. در پایتون، `True` معادل `1` و `False` معادل `0` در نظر گرفته می‌شود. از آنجایی که `1` و `0` قبلا در مجموعه قرار گرفته‌اند، `True` و `False` به عنوان اعضای تکراری در نظر گرفته شده و اضافه نمی‌شوند.

my_function(fname = 'Tobias', lname = 'Refsnes')
def my_function(**kwargs):
  print('His last name is ' + kwargs['lname'])
His last name is Refsnes
His last name is Tobias
SyntaxError: invalid syntax

۶ خروجی کد زیر چیست؟ 

thislist = ['apple', 'banana', 'cherry', 'orange', 'kiwi', 'mango']
thislist[1:3] = ['blackcurrant', 'watermelon', 'grape']
print(thislist)
['apple', 'blackcurrant', 'watermelon', 'grape', 'orange', 'kiwi', 'mango']
['apple', 'blackcurrant', 'watermelon', 'cherry', 'orange', 'kiwi', 'mango']
apple', 'banana', 'cherry','blackcurrant', 'watermelon', 'grape', 'orange','kiwi', 'mango'
ValueError: attempt to assign sequence of size 3 to slice of size 2
توضیح: این کد یک جایگزینی برشی (slice assignment) انجام می‌دهد. `thislist[1:3]` به اعضای ایندکس ۱ و ۲ (یعنی `'banana'` و `'cherry'`) اشاره دارد. این دو عضو با لیست جدید جایگزین می‌شوند. در این نوع جایگزینی، طول لیست جایگزین می‌تواند متفاوت از طول برشی باشد که حذف می‌شود.

۷ کدام یک از گزینه‌های زیر باعث خطا می‌شود؟

print(True + 1)
print(int('۲.۵'))
print(float('۱'))
print(str(1) + str(2))
توضیح: تابع `int()` نمی‌تواند یک رشته که نمایانگر عدد اعشاری است را مستقیما به عدد صحیح تبدیل کند. این کار باعث بروز خطای `ValueError` می‌شود. برای تبدیل، ابتدا باید آن را به `float` و سپس به `int` تبدیل کرد: `int(float('2.5'))`.

۸ خروجی کد زیر چیست؟ 

list1 = [1, 2, 3]
list2 = list1
list2 = [4, 5, 6]
print(list1)
[1, 2, 3]
[4, 5, 6]
[1, 2, 3, 4, 5, 6]
Error: UnboundLocalError
توضیح: ابتدا `list2` به همان شیء لیستی اشاره می‌کند که `list1` به آن اشاره دارد. اما در خط سوم، یک شیء لیست جدید `[4, 5, 6]` ساخته می‌شود و نام `list2` به این شیء جدید اختصاص داده می‌شود (re-assignment). این کار هیچ تاثیری روی شیء اصلی که `list1` به آن اشاره دارد، نمی‌گذارد.

۹ خروجی کد زیر چیست؟ 

text = 'hello'
text2 = text.replace('l', 'p').upper().strip()
print(text2)
HEPPO
heppo
HEPPO WORLD
hello
توضیح: این یک نمونه از زنجیره‌سازی متدها (method chaining) است:
۱. `text.replace('l', 'p')` رشته را به `'heppo'` تبدیل می‌کند.
۲. `.upper()` نتیجه را به `'HEPPO'` تبدیل می‌کند.
۳. `.strip()` فواصل خالی ابتدا و انتها را حذف می‌کند که در اینجا تاثیری ندارد.

۱۰ خروجی کد زیر چیست؟ 

txt = 'hello world'
if 'world' in txt or 'python' not in txt:
    print('True')
else:
    print('False')
True
False
SyntaxError
TypeError
توضیح: شرط `if` دارای یک عبارت `or` است. برای اینکه کل شرط `True` شود، کافیست فقط یکی از دو طرف `True` باشد.
۱. عبارت اول `'world' in txt` بررسی می‌شود که نتیجه‌اش `True` است.
۲. چون بخش اول `True` است، پایتون دیگر بخش دوم را بررسی نمی‌کند (short-circuiting) و بلافاصله بلوک `if` را اجرا می‌کند.
